# FROSTBORNE DEADLANDS — сайт релиза

Полностью готовый статический сайт. Чистый HTML + CSS + JS, без сборки и зависимостей сборщика.
Работает локально (открыть `index.html`) и на любом статическом хостинге.

## 🖼️ Какие картинки куда вставить

| Файл | Что снять | Размер | Где видно |
|---|---|---|---|
| `hero-bg.webp` | Широкий кадр заснеженного разрушенного мира | 1920×1080 | Фон первого экрана |
| `shader-before.webp` | Кадр БЕЗ шейдеров | 1600×900 | Левая сторона слайдера |
| `shader-after.webp` | ТОТ ЖЕ кадр С шейдерами | 1600×900 | Правая сторона слайдера |
| `shot-1…6.webp` | Атмосферные сцены | 1280×800 | Галерея (лайтбокс) |

**Как сохранить в WebP:** в любом редакторе «Экспорт → WebP, качество ~82%», либо онлайн-конвертер PNG→WebP.
WebP весит в 3–4 раза меньше PNG при том же качестве — это даёт Lighthouse 90+.

Если хочешь оставить PNG — просто переименуй файлы и поменяй расширения в `index.html`
(поиск по `.webp`), но WebP предпочтительнее.

## 🔥 Firebase (живая статистика просмотров / скачиваний / онлайна)

Конфиг уже вставлен в `firebase-config.js`. Чтобы статистика заработала «вживую»:

1. Зайди в [Firebase Console](https://console.firebase.google.com/) → проект `frostborne-1ca0b`.
2. **Realtime Database** → если не создана, создай (регион любой, ближе к аудитории).
3. Вкладка **Rules** → на время теста поставь:
   ```json
   {
     "rules": {
       "counters": { ".read": true, ".write": true },
       "presence": { ".read": true, ".write": true }
     }
   }
   ```
   > Это открытые правила (любой может писать счётчики). Для публичного релиза этого достаточно —
   > красть там нечего. Если хочешь строже — ограничь запись только инкрементом через Cloud Functions.
4. Готово. Сайт сам создаст `counters/views`, `counters/downloads` и `presence/*`.

Без этих шагов сайт работает в **DEMO-режиме**: счётчики живут локально в браузере. Всё остальное (снег, звук, шейдер-слайдер, режимы) работает всегда.

## 🌐 Как сделать, чтобы сайт работал ПОСТОЯННО (варианты хостинга)

Все варианты ниже — **бесплатные** и дают HTTPS из коробки.

### Вариант 1 — GitHub Pages (рекомендую, самый простой)
1. Создай репозиторий на GitHub, например `frostborne-deadlands`.
2. Залей туда всё содержимое папки `site/` (index.html в корень).
3. Settings → Pages → Source: `Deploy from a branch` → ветка `main`, папка `/root`.
4. Через минуту сайт будет на `https://ТВОЙ_НИК.github.io/frostborne-deadlands/`.
5. (Опц.) Привяжи свой домен в Settings → Pages → Custom domain.

### Вариант 2 — Netlify (drag & drop, без git)
1. Зайди на [netlify.com](https://netlify.com), зарегистрируйся.
2. Просто перетащи папку `site/` в окно «Deploy» — всё.
3. Получишь адрес вида `https://frostborne.netlify.app` (имя меняется в настройках).
4. Свой домен подключается в 2 клика.

### Вариант 3 — Cloudflare Pages (самый быстрый CDN)
1. [pages.cloudflare.com](https://pages.cloudflare.com) → Create project.
2. Подключи GitHub-репо ИЛИ загрузи папку напрямую.
3. Build command: оставь пустым. Output directory: `/`.
4. Готово — раздаётся с глобального CDN Cloudflare (очень быстро в РФ/СНГ).

### Вариант 4 — Vercel
1. [vercel.com](https://vercel.com) → Import репозиторий или Deploy папку.
2. Framework Preset: `Other`. Деплой.

> Для **домена** (например `frostborne.ru`): купи на reg.ru / nic.ru / Cloudflare,
> в DNS добавь записи, которые подскажет выбранный хостинг. У всех четырёх это занимает ~5 минут.

- **🔥 Костёр** — форум: «Предложения и идеи» (с апвоутами, лучшие всплывают наверх), «Технический отсек» (баг-трекер), «FAQ» (раскрывающиеся спойлеры).

**Профиль игрока:** позывной (ник Minecraft → подтягивает скин), статус («Замерзает», «Ищет команду», «Техномаг» или свой). Хранится локально + в Firebase presence.

### Firebase-схема для Эфира

Realtime Database заполнится сама. Для работы добавь в **Rules** путь `aether`:
```json
{
  "rules": {
    "counters":  { ".read": true, ".write": true },
    "presence":  { ".read": true, ".write": true },
    "chronicle": { ".read": true, ".write": true },
    "aether": {
      ".read": true,
      "chat":     { ".write": true, "$m": { ".validate": "newData.hasChildren(['nick','msg','ts'])" } },
      "ideas":    { ".write": true },
      "bugs":     { ".write": true },
      "presence": { ".write": true }
    }
  }
}
```

> ⚠️ **Модерация:** чат публичный и анонимный. Встроены базовые защиты — лимит длины (500), антифлуд (1 сообщение / 1.5 c), мягкий фильтр мата, экранирование HTML (защита от XSS). Но полную модерацию (бан, удаление) лучше добавить отдельно — например, через простую админ-страницу или Firebase Cloud Functions. Картинки по ссылкам игроков рендерятся как есть — следи за контентом или отключи рендеринг изображений, если будут злоупотребления.

## ☠ Как подключить «Хроники погибших» к реальной статистике

«Хроники» работают на том же Firebase, что и счётчики. Чтобы цифры были **общими для всех игроков** (а не локальными в браузере):

1. Открой [Firebase Console](https://console.firebase.google.com/) → проект `frostborne-1ca0b` → **Realtime Database**.
2. Вкладка **Rules** — убедись, что путь `chronicle` открыт на чтение/запись:
   ```json
   {
     "rules": {
       "counters":  { ".read": true, ".write": true },
       "presence":  { ".read": true, ".write": true },
       "aether":    { ".read": true, ".write": true },
       "chronicle": {
         ".read": true,
         "total":   { ".write": true },
         "record":  { ".write": true },
         "causes":  { ".write": true },
         "deaths":  { ".write": true }
       }
     }
   }
   ```
3. Готово. Структура создастся сама при первой записи:
   - `chronicle/total` — общее число смертей (число)
   - `chronicle/causes/{Причина}` — счётчик по каждой причине
   - `chronicle/record` — `{day, nick}` рекордсмена
   - `chronicle/deaths` — лента записей `{nick, day, cause, ts}`

Когда игрок жмёт «Зафиксировать гибель» и заполняет карточку — Firebase атомарно (через транзакции) увеличивает счётчики, и **у всех на сайте цифры обновляются в реальном времени**.

Без настройки правил «Хроники» работают в **демо-режиме** (видны только тебе, с примерами).

> ⚠️ Открытые правила (`.write: true`) означают, что любой может писать в базу — для публичного сайта-визитки это нормально (красть нечего, накрутка счётчика смертей некритична). Если захочешь защиту от спама — добавь Firebase App Check или Cloud Functions с валидацией.
